# 番茄短篇AI创作工具 - 技术推进与架构指南

本项目是一个面向番茄小说平台的短篇小说AI辅助创作工具。核心理念：**AI当编剧引导用户把创意结构化**，而非AI替用户写。

## 🪐 核心开发原则

1. **结构化优先**：所有创作动作（梗概、扩写、润色）必须基于“节奏点”和“约束标签”。
2. **渐进式生成**：从原始灵感 -> 6步向导（类型/节奏/角色/走向/大纲/导语）-> 章节扩写。
3. **番茄适配**：严格遵循番茄平台“前压后弹”的黄金节奏律（150字钩子，每1500字一个情绪转折）。

## 🛠️ 技术架构

### 后端 (Python FastAPI + SQLAlchemy + SQLite)
- **数据库**：单文件 `app.db`，异步访问。
- **Provider**：支持 OpenAI 兼容协议。内置主/备 API 切换逻辑。
- **服务层**：
    - `TextAIService`：统一大模型交互入口。
    - `PromptService`：管理 13+ 个内置创作模板。
    - `ProjectWizard`：维持 1-6 步的状态机。

### 前端 (React + Ant Design + Vite)
- **核心组件**：
    - `ProjectList`：支持多选番茄类别，书名/简介校验。
    - `ProjectWizard`：SSE 流式展示或分步确认界面。
    - `ProjectEditor`：双栏高亮联动编辑器，左侧为情绪节奏线，右侧为创作区。

## 📝 核心业务资产

### 1. 节奏约束体系 (Rhythm Constraints)
在调用 AI 生成内容时，必须强制注入以下标签：
- **情绪类**：憋屈蓄力、小释放、大释放、惊愕反转、虐、暖。
- **信息类**：禁止揭示真相、只给读者信息差、埋伏笔、回收伏笔。
- **技巧类**：欲扬先抑、打脸、断章钩子。

### 2. 番茄导语公式
`导语 = 处境速写(≤30%) + 冲突爆发(≥40%) + 悬念钩子(20-30%)`
字数控制：150-300字最佳。

### 3. 类型/题材映射 (Genre)
本项目已预设 20+ 个番茄平台原生类型，支持多选存储（逗号分隔）。

### 4. API 激活策略 (Activation Strategy)
- **单核模式 (Singleton)**：同一类型（Text/Image）的配置中，必须且只能有一个处于 `is_active` 状态。
- **互斥联动**：在前端/后端更新 `is_default` 时，必须自动将同类型的其他记录标记为不可用。
- **安全脱敏**：列表页严禁暴露完整 API KEY，仅支持前 8 位明文展示。
